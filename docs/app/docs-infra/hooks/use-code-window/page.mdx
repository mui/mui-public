# Use Code Window

The `useCodeWindow` hook layers code-block-specific choreography on top of [`useScrollAnchor`](../use-scroll-anchor/page.mdx) so a syntax-highlighted snippet can expand and collapse without the user losing their place — and without the horizontal scrollbar snapping in or out.

It is the recommended hook when you're building a **collapsible code block**. For non-code layouts, reach for [`useScrollAnchor`](../use-scroll-anchor/page.mdx) directly.

## What it does

On top of `useScrollAnchor`'s page-scroll compensation, this hook:

1. **Picks the anchor for you.** Looks for an element matching `[data-frame-type="highlighted"], [data-frame-type="focus"]` inside the container (the convention used by the highlighter pipeline). Falls back to the toggle ref when no such frame exists, or when the highlighted frame is offscreen on collapse.
2. **Animates the scrollbar gutter.** Toggles a `data-scrollbar-gutter` attribute on the inner `<pre>` so your CSS can swap between a real horizontal scrollbar and equivalent `padding-bottom` instead of snapping when overflow appears or disappears.
3. **Slides scroll back to column 0 on collapse.** Translates the inner `<code>` element back to `scrollLeft = 0` via a compositor-driven WAAPI animation, so the focused region (which usually starts at the left edge) is back in view after collapsing. Honors `prefers-reduced-motion`.
4. **Cancels per-`<pre>` animations on unmount.**

```tsx
import { useCodeWindow } from '@mui/internal-docs-infra/useCodeWindow';

function CollapsibleSnippet() {
  const [expanded, setExpanded] = React.useState(false);
  const { containerRef, toggleRef, anchorScroll } = useCodeWindow<HTMLButtonElement>();

  return (
    <div ref={containerRef}>
      <pre className={expanded ? 'expanded' : ''}>
        <code>{source}</code>
      </pre>
      <button
        ref={toggleRef}
        type="button"
        onClick={() => {
          // Anchor before the layout-changing state update.
          anchorScroll(expanded ? 'collapse' : 'expand');
          setExpanded((prev) => !prev);
        }}
      >
        {expanded ? 'Collapse' : 'Expand'}
      </button>
    </div>
  );
}
```

The hook expects a structure of `<div ref={containerRef}><pre><code>…</code></pre>…</div>`. The `<pre>` and `<code>` elements are how the gutter and scroll-back animations find their target.

## Configuration

Defaults match the highlighter's CSS conventions, but every opinion is configurable:

```tsx
useCodeWindow({
  expandDuration: 350, // ms — must match your CSS expand transition
  collapseDuration: 350, // ms — must match your CSS collapse transition
  scrollBackDuration: 300, // ms — set to 0 to disable
  anchorSelector: '[data-frame-type="highlighted"], [data-frame-type="focus"]',
  collapsibleProbeSelector: '[data-collapsible]',
});
```

### `expandDuration` / `collapseDuration`

Should equal the CSS transition duration on whichever property drives the height change (`height` with `interpolate-size`, `max-height`, etc.). The hook keeps the page-scroll compensation alive for this long, then hands control back to the user.

### `scrollBackDuration`

The horizontal `<code>`-transform animation that returns the snippet to column 0 on collapse. The default `300` matches the gutter animation; set to `0` to disable.

### `anchorSelector`

Selector evaluated inside the container to pick the anchor. The first match wins. Override this if your markup uses different data attributes.

### `collapsibleProbeSelector`

When set, the expand transition only animates the gutter if at least one element matching this selector exists inside the `<pre>`. This avoids unnecessary attribute toggling for plain (non-truncated) snippets where no scrollbar will appear after expansion.

## CSS contract

The hook flips a `data-scrollbar-gutter` attribute on the inner `<pre>`. To get the gutter animation, your stylesheet must provide the corresponding rules:

```css
/* Just before collapse: lock overflow and reserve the scrollbar's space */
pre[data-scrollbar-gutter='collapse-from'] {
  overflow-x: hidden;
  padding-bottom: var(--scrollbar-h);
}
/* Animate to the resting padding */
pre[data-scrollbar-gutter='collapse-to'] {
  overflow-x: hidden;
  padding-bottom: 12px;
  transition: padding-bottom 0.35s ease;
}
/* Mirror for expand if you want the scrollbar to fade in */
pre[data-scrollbar-gutter='expand-from'] {
  overflow-x: hidden;
  padding-bottom: 12px;
}
pre[data-scrollbar-gutter='expand-to'] {
  overflow-x: hidden;
  padding-bottom: var(--scrollbar-h);
  transition: padding-bottom 0.35s ease;
}
```

Without these rules the attribute is harmless — it just toggles silently.

### Indent Shifting

With no padding configured, only highlighted/focus and normal frames are produced. Opt in to the `data-frame-indent` attribute on region frames by passing `emitFrameIndent: true` to `createEnhanceCodeEmphasis` — CSS can then read it to know how far the code is indented. This demo highlights an import statement (indent 0) and uses standalone `@focus-start`/`@focus-end` around the deeply nested `<DatePicker>` usage (indent 3). When collapsed, the focused frame shifts left by `6ch` so it aligns with the left edge, then resets when expanded.

### Focus Max Size

When `focusFramesMaxSize` is set and a highlighted region exceeds that limit, a focused window is taken from the start of the region, and the remaining overflow lines are marked as unfocused. This demo uses `focusFramesMaxSize: 6` with the `<form>` JSX highlighted (10 lines). When collapsed, only the first 6 lines of the region are visible; expanding reveals the full highlighted region with the overflow lines.
